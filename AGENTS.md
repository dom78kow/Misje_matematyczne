# AGENTS.md

# Project Overview

This repository contains the educational project "Misje Matematyczne".

The project is intended for students and teachers. Every change should preserve educational quality, readability and consistency.

The primary language of all documentation is Polish.

---

# Goals

When making changes:

- Preserve educational value.
- Keep the narrative style.
- Do not simplify mathematical content unless explicitly requested.
- Do not introduce factual errors.
- Keep terminology consistent across all missions.

---

# Documentation

Use Markdown.

Prefer:

- short sections
- descriptive headings
- bullet lists where appropriate
- tables for structured information

Avoid excessive formatting.

---

# Mission Structure

Each mission should follow the existing repository structure.

When creating a new mission:

- follow previous mission layout
- keep naming conventions
- keep numbering consistent
- update indexes when required

Never invent a different structure.

---

# Educational Content

Never modify exercises unless explicitly requested.

Suggestions are allowed, but existing tasks should remain unchanged.

When generating new exercises:

- increase the difficulty level gradually in successive missions
- keep language suitable for the target age
- avoid ambiguity
- provide clear instructions

---

# Writing Style

Use:

- clear Polish
- simple language
- consistent terminology

Avoid:

- unnecessary repetition
- long paragraphs
- informal language

---

# Repository Rules

Before creating new files:

- check if similar material already exists
- avoid duplication

Before deleting files:

- ask for confirmation.

Never rename folders without permission.

---

# Git

Prefer small logical commits.

Suggested commit messages:

docs:
feat:
fix:
refactor:

---

# Pull Requests

When preparing a Pull Request:

- summarize changes
- explain why changes were made
- mention affected missions
- list files modified

---

# Agent Behaviour

Unless explicitly requested:

- do not rewrite entire documents
- do not change repository organization
- do not remove educational material
- do not translate documentation
- do not modify licenses

Always explain significant changes before applying them.
